# User Rules - قوانین شخصی مهدی باغبان‌پور

## زبان و ارتباط
- همه توضیحات و کامنت‌ها باید به **فارسی** باشند
- پیام‌های commit به فارسی و با emoji مناسب
- متغیرها و نام‌های تابع به انگلیسی با camelCase

## استاندارد کدنویسی
- استفاده از 2 space برای indentation
- استفاده از single quote (') برای string ها در JavaScript
- همیشه async/await به جای promise chain
- error handling کامل با try-catch

## فلسفه توسعه
- کد باید clean و readable باشد
- هر تابع یک کار انجام بدهد (Single Responsibility)
- کامنت‌های توضیحی برای logic های پیچیده
- همیشه validation در سمت frontend و backend

## اولویت‌ها
1. امنیت (Security First)
2. تجربه کاربری (UX)
3. کارایی (Performance)
4. قابلیت نگهداری (Maintainability)

## توضیحات
- قبل از هر تغییر، توضیح بده چرا این تغییر لازمه
- بعد از هر کد، بگو چه مشکلی حل می‌شه
- آموزش مفهوم‌های جدید با مثال

## تست و دیباگ
- همیشه console.log برای debugging
- تست در Chrome و Firefox حتماً
- بررسی RTL و LTR
